AI Devtools · 29 May 2026

Anthropic ships an agentic build pipeline

847 words· 6-min read· 14 sources· salience .82 ↑ EN|ไทย

Key points

  • Closed-loop agents plan, write, and verify against a test harness before opening a PR.
  • ~40% cheaper per task via prompt caching + planner→Opus routing only on hard steps.
  • Green-field wins, brown-field still needs a babysitter on legacy codebases.
Why it matters

The first credible "describe → ship" loop a small team can run unattended — directly relevant to how NDF DEV scaffolds internal tools.

Anthropic's new pipeline closes the loop that earlier coding agents left open: instead of emitting a diff and hoping, the agent now runs the project's own test harness, reads the failures, and iterates until green before a human ever sees a pull request.1 That single change moves the trust boundary from "review every line" to "review the outcome."

The trust boundary moves from every line to the outcome.

The news

The release pairs a cheap planner model with selective routing to Opus on the hard steps, which is where the ~40% cost reduction comes from.2 Prompt caching across the plan-act-verify cycle does the rest.

Signals

Early adopters split cleanly by codebase age. Green-field projects report near-autonomous runs; brown-field teams still report a babysitter tax on anything touching legacy modules and implicit conventions.3
